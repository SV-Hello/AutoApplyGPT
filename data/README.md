# Data Directory

This directory contains the datasets used for training AutoApplyGPT.

## Directory Structure

```
data/
├── resumes/          # Original resume dataset
├── jobs/             # Original job postings dataset
└── processed/        # Processed and generated training data
```

## Required Datasets

### 1. Resume Dataset
- **Source:** [UpdatedResumeDataSet.csv on Kaggle](https://www.kaggle.com/datasets/snehaanbhawal/resume-dataset)
- **Size:** 3,446 resumes
- **Download:** Download `Resume.csv` and place in `data/resumes/`

### 2. Job Postings Dataset  
- **Source:** [LinkedIn Job Postings on Kaggle](https://www.kaggle.com/datasets/arshkon/linkedin-job-postings)
- **Size:** 123,849 job postings
- **Download:** Download `postings.csv` and place in `data/jobs/`
- **Note:** Rename to `postings1.csv` if needed

## Setup Instructions

1. **Download the datasets:**
   ```bash
   # Download from Kaggle (requires Kaggle account)
   kaggle datasets download -d snehaanbhawal/resume-dataset
   kaggle datasets download -d arshkon/linkedin-job-postings
   ```

2. **Extract and place files:**
   ```bash
   # Unzip and move to appropriate folders
   unzip resume-dataset.zip
   mv Resume.csv data/resumes/
   
   unzip linkedin-job-postings.zip
   mv postings.csv data/jobs/postings1.csv
   ```

3. **Verify setup:**
   ```bash
   # Check files are in place
   ls data/resumes/Resume.csv
   ls data/jobs/postings1.csv
   ```

## Processed Data

The `processed/` directory will contain:
- `clean_resumes_tech.csv` - Filtered technical resumes
- `clean_jobs_tech.csv` - Filtered technical job postings
- `full_pairs_dataset.csv` - Resume-job pairs with similarity scores
- `training_data.jsonl` - Generated training examples (resume + cover letter pairs)
- `training_data_filtered.jsonl` - Quality-filtered training data

These files are generated by running the pipeline scripts in order.

## Data Statistics

**After filtering (tech/engineering only):**
- Resumes: ~150-1,300 (varies by filtering strictness)
- Jobs: ~250-93,000 (varies by filtering strictness)
- Training pairs: User-configurable (default: 2,000-10,000)

## Privacy & Usage

- Original datasets are publicly available on Kaggle
- All resumes in the dataset are anonymized
- Job postings are from public LinkedIn listings
- Use datasets in accordance with Kaggle's terms of service
- Generated training data is for research/educational purposes
